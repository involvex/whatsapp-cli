# WhatsApp CLI - Fix Reconnection & Chat History Issues

## User Requirements

1. **Browser "already running" fix** - Clean up stale lock files on restart
2. **Chat history must persist across sessions** - CRITICAL: Chats/messages should be saved and restored
3. **Fix history fetch crash** - Handle `waitForChatLoading` TypeError gracefully
4. **Improve UI indicators** - Show clear connection states

### 1. "Browser is already running" Error on Restart

**Root Cause**: When the CLI crashes or is forcefully terminated, Puppeteer leaves behind a lock file (`auth/session/lockfile`). On restart, Puppeteer detects this and refuses to start, causing the error.

**Location**: `src/client.ts` - `initializeClient()` function

**Solution**:

- Add a function to clean up stale browser lock files before initialization
- Check for and remove `lockfile` and `SingletonSocket` files in the session directory
- Add a `--kill-browser` or `--force` CLI option for manual cleanup
- Auto-detect and clean stale sessions on startup with a warning

### 2. Chat History Persistence (CRITICAL)

**Current State**: Config has `chatHistoryEnabled: true` and `PATHS.chatHistory` defined, but nothing saves/restores chat data.

**Solution**:

- Create a `chatPersistence.ts` module to handle save/load of chats and messages
- On client `ready`: Load cached chats + messages from `chat-history.json`
- On new messages: Append to cache and save to disk periodically
- On app start: Restore last known chats and messages before WhatsApp syncs
- Use incremental updates (append only new messages) to avoid slow writes

### 3. Chat History Fetch Crash (`waitForChatLoading` TypeError)

**Root Cause**: The `chat.fetchMessages()` method in whatsapp-web.js relies on WhatsApp Web's internal `waitForChatLoading` function. WhatsApp Web periodically updates their frontend, breaking this compatibility.

**Location**: `src/cli.tsx` lines 115-139 and 155-171

**Solution**:

- Add try-catch around `fetchMessages()` calls
- Display a user-friendly error when history fetch fails
- Add a manual refresh option that retries with exponential backoff
- Optionally: Show "History unavailable" message instead of crashing

### 4. UI Indicators Not Clear

- Add connection state enum: `disconnected` | `connecting` | `ready` | `loading_history`
- Update Footer/Sidebar to show current state
- Add visual feedback for all actions
